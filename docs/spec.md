# タスク管理アプリ 仕様書

## 概要

Next.js 16 (App Router) + Prisma 5 + SQLite で構築したタスク管理 Web アプリ。
スプリント・エピック・プロジェクトを組み合わせたアジャイル的な運用を想定。

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Next.js 16 App Router |
| UI ライブラリ | React 19（named imports のみ、`React.xxx` 不可） |
| スタイリング | Tailwind CSS |
| ORM | Prisma 5 |
| DB | SQLite（`prisma/dev.db`） |
| DnD | @dnd-kit |

## データモデル

### Project（プロジェクト）

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| name | String | プロジェクト名 |
| key | String? | ユニークなプレフィックス（例: `PROJ`）。課題キー生成に使用 |
| description | String? | 説明 |
| color | String | カラー（デフォルト: `#6366f1`） |
| status | String | `active`（デフォルト）/ `closed` |
| archivedAt | DateTime? | アーカイブ日時（null = 非アーカイブ） |
| createdAt | DateTime | 作成日時 |

### Epic（エピック）

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| title | String | タイトル |
| description | String? | 説明 |
| status | String | `open` / `closed` |
| color | String | カラー（デフォルト: `#818cf8`） |
| projectId | Int? | FK → Project（SetNull on delete） |
| order | Int | 表示順 |
| createdAt | DateTime | 作成日時 |

### Sprint（スプリント）

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| name | String | スプリント名 |
| startDate | DateTime? | 開始日 |
| endDate | DateTime? | 終了日 |
| status | String | `planning` / `active` / `completed` |
| order | Int | 表示順 |
| closedAt | DateTime? | クローズ日時（completed 時に記録） |
| createdAt | DateTime | 作成日時 |

### SprintTaskRecord（スプリント完了時タスク記録）

スプリント完了時に親タスクのスナップショットを記録。タスク削除後も履歴として残す。

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| sprintId | Int | FK → Sprint（Cascade） |
| taskId | Int | タスクID（FK なし、タスク削除後も保持） |
| taskTitle | String | 完了時点のタイトル |
| wasDone | Boolean | 完了していたか（`done=true` または `status=closed`） |
| movedToSprintId | Int? | 移動先スプリントID（未完了タスクのみ） |
| movedToSprintName | String? | 移動先スプリント名（未完了タスクのみ） |
| createdAt | DateTime | 記録日時 |

### Task（タスク）

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| title | String | タイトル |
| description | String? | 説明（Markdown） |
| status | String | `open` / `in_progress` / `resolved` / `on_hold` / `closed` |
| done | Boolean | 完了チェック（デフォルト: false） |
| priority | String | `low` / `medium` / `high` |
| dueDate | DateTime? | 期限 |
| sprintId | Int? | FK → Sprint（null = バックログ） |
| projectId | Int? | FK → Project |
| taskNumber | Int? | プロジェクト内連番（課題キー生成に使用） |
| order | Int | 表示順 |
| createdAt | DateTime | 作成日時 |
| epicId | Int? | FK → Epic（SetNull on delete） |
| parentId | Int? | FK → Task（サブタスク用、Cascade） |

### TaskActivity（タスクアクティビティ）

タスクへの変更操作を自動記録。

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| taskId | Int | FK → Task（Cascade） |
| field | String | 変更項目名 |
| oldValue | String? | 変更前の値（人間が読める形式） |
| newValue | String? | 変更後の値（人間が読める形式） |
| createdAt | DateTime | 記録日時 |

**記録される操作:**

| field | oldValue | newValue | トリガー |
|-------|----------|----------|---------|
| タイトル | 旧タイトル | 新タイトル | タイトル変更 |
| 説明 | `"あり"` / null | `"更新"` / null | 説明変更 |
| ステータス | 旧ラベル | 新ラベル | ステータス変更 |
| 完了 | `"完了"` / `"未完了"` | `"完了"` / `"未完了"` | done トグル |
| 優先度 | 旧ラベル | 新ラベル | 優先度変更 |
| 期限 | 旧日付（YYYY-MM-DD） | 新日付 | 期限変更 |
| スプリント | 旧スプリント名 / null | 新スプリント名 / null | スプリント変更 |
| プロジェクト | 旧プロジェクト名 / null | 新プロジェクト名 / null | プロジェクト変更 |
| エピック | 旧エピック名 / null | 新エピック名 / null | エピック変更 |
| コメントを追加 | null | コメント本文（最大80字） | コメント投稿 |
| コメントを編集 | 編集前本文（最大80字） | 編集後本文（最大80字） | コメント編集 |
| コメントを削除 | 削除本文（最大80字） | null | コメント削除 |
| サブタスクを追加 | null | サブタスクタイトル | サブタスク作成 |
| サブタスクを削除 | サブタスクタイトル | null | サブタスク削除 |

### Comment（コメント）

| カラム | 型 | 説明 |
|--------|----|------|
| id | Int | PK |
| body | String | 本文（Markdown） |
| taskId | Int | FK → Task（Cascade） |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時（自動） |

## 課題キー

プロジェクトに `key`（例: `PROJ`）が設定されている場合、タスクに課題キーが付与される。

```
{project.key}-{task.taskNumber}  →  例: PROJ-42
```

- `taskNumber` はプロジェクト内で連番採番（PATCH でプロジェクト変更時も再採番）
- プロジェクト未設定のタスクには課題キーなし
- 課題キーはタスクカード・詳細ビュー・右パネルに表示
  - **左クリック**: SPA ナビゲーションでそのタスクの詳細ビューへ遷移
  - **右クリック / 中クリック**: ブラウザ標準の `<a href="/tasks/{id}">` によりブラウザの新しいタブで `/tasks/{id}` ページを開く

## 画面構成

### グローバルヘッダー

画面最上部に常時表示。

```
[タスク管理]  ← タイトル
              [タスク] [スプリント]  ← メニュータブ（pill スタイル）
                                    [+ タスクを追加]  [⚙]  ← 右端
```

- **「タスク」「スプリント」「プロジェクト」タブ**: `mainMenu` ステートを切り替える。選択中はホワイト背景
- **「+ タスクを追加」**: 右パネルに新規作成フォームを開く（常時有効）
- **⚙（歯車アイコン）**: 管理メニュー（Admin モーダル）を開く

## タスク画面（mainMenu === "tasks"）

### 全体レイアウト

```
┌──────────────────────────────────────────────────────┐
│ グローバルヘッダー                                      │
├──────────┬──────────────────────────────┬────────────┤
│          │ コンテンツヘッダー             │            │
│ サイドバー│  ビュータイトル・フィルター   │  右パネル   │
│  w-56   │  リスト/スイムレーン 切替      │   w-80     │
│（折り畳み│                              │（タスク詳細 │
│  可）   │ メインコンテンツ               │  / 作成）  │
│         │                              │            │
└──────────┴──────────────────────────────┴────────────┘
```

- サイドバーは折り畳み可能（`w-56` ↔ `w-10`、トグルボタンで切替）
- 右パネルは `isTaskView(view)` のとき非表示（タスク詳細ビューが全幅を使う）

---

### サイドバー

3 セクション構成。折り畳み時は全セクション非表示（トグルボタンのみ表示）。

#### プロジェクト セクション

- **表示対象**: `status === "active"` かつ `archivedAt === null` のプロジェクトのみ（終了・アーカイブ済みは非表示）
- プロジェクト一覧（色ドット + 名前 + 親タスク数）
- クリックでプロジェクトフィルター ON/OFF（選択中はインディゴ背景）
- 「+ 追加」ボタン → インラインフォームを展開
  - 入力項目: プロジェクト名（必須）、キー（英数大文字、最大15文字）、カラー（8色から選択）
- プロジェクトにタスクをドロップすると `projectId` を変更（`DroppableProjectItem`）
- 各プロジェクト行にホバーで「編集（鉛筆）」ボタンのみ表示
- 編集・終了・アーカイブ・削除はすべて編集モーダル（後述）から操作

#### エピック セクション

- 「エピックビュー」ボタン: クリックで `view` を `"all-epics"` に切替（選択中はインディゴ背景）
- エピック一覧（色スクエア + タイトル + 親タスク数）
- クリックでエピックフィルター ON/OFF

#### スプリント セクション

- 「スプリントビュー」ボタン: クリックで `view` を `"all-sprints"` に切替（選択中はインディゴ背景）
- スプリント総数を表示

---

### コンテンツヘッダー（タスク詳細ビュー以外で表示）

```
スプリントビュー  [プロジェクトA ×] [エピックB ×]    [リスト] [スイムレーン]
```

- ビュータイトル（「スプリントビュー」または「エピックビュー」）
- アクティブなフィルターをチップ表示（クリックで解除）
  - プロジェクトフィルター: プロジェクト色 + 名前 + ×
  - エピックフィルター: エピック色 + 名前 + ×
- **リスト / スイムレーン 切替**（`displayMode` ステート）

---

### スプリントビュー・リスト表示（view === "all-sprints" かつ displayMode === "list"）

スプリント一覧とバックログをリスト形式で表示。

#### スプリント追加

- 「+ スプリントを追加」ボタン → インラインフォームを展開
  - 入力項目: スプリント名（必須）・開始日・終了日・ステータス
  - 「追加」「キャンセル」ボタン

#### スプリントセクション（SortableSprintSection）

非 completed スプリントをセクション単位で縦並び表示。セクション自体もドラッグで並び替え可能。

各セクションのヘッダー:
- スプリント名
- 開始日・終了日（設定されている場合）
- ステータスバッジ（planning / active）
- タスク数（完了数/総数）
- 「完了」ボタン（スプリント完了モーダルを開く）
- 「削除」ボタン
- 編集ボタン（スプリント編集モーダルを開く）

セクション本体:
- タスクカード（`DraggableTaskCard`）のリスト
- タスクがない場合は「タスクなし」メッセージ
- ドラッグ中に他スプリントのタスクが上を通ると枠線をハイライト

#### バックログ（MainBacklogDropZone）

スプリント一覧の最下部に常時表示。

```
┌──────────────────────────────────┐
│ ■ バックログ                N 件 │  ← ヘッダー
├──────────────────────────────────┤
│  タスクカード...                  │
└──────────────────────────────────┘
```

- `sprintId === null` かつ `parentId === null` のタスクを表示
- プロジェクト・エピックフィルターが適用される
- ドラッグ中にホバーすると破線枠にハイライト、「ここにドロップしてバックログへ移動」メッセージ
- バックログ内でもタスクカードの並び替え可能

---

### エピックビュー・リスト表示（view === "all-epics" かつ displayMode === "list"）

エピックごとにタスクをグループ化して表示。

#### エピック追加

- 「+ エピックを追加」ボタン → インラインフォームを展開
  - 入力項目: エピック名（必須）・プロジェクト（任意）・カラー（パレットから選択）

#### エピックセクション（EpicDropZone）

各エピックのセクション:
- ヘッダー: エピック色スクエア + タイトル + プロジェクト名 + タスク数 + 編集・削除ボタン
- ヘッダークリック: エピックフィルターを適用してスプリントビューへ切替
- タスクカード（`DraggableTaskCard`）のリスト
- タスクがない場合は「ストーリーなし」メッセージ
- ドラッグ中にホバーすると破線枠にハイライト

**「エピックなし」セクション**（最下部）:
- `epicId === null` のタスクを表示
- 破線枠スタイル

### スイムレーンビュー（displayMode === "swimlane"）

ステータス別に5カラム構成。**アクティブなスプリント（`status === "active"`）のタスクのみ表示。**

```
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  オープン│  着手   │ 解決済み│  保留   │クローズ │
│         │         │         │         │         │
│ タスク  │ タスク  │ タスク  │ タスク  │ タスク  │
│ カード  │ カード  │ カード  │ カード  │ カード  │
└─────────┴─────────┴─────────┴─────────┴─────────┘
```

- アクティブスプリントがない場合は「アクティブなスプリントがありません」メッセージを表示
- 複数のアクティブスプリントがある場合は全スプリントのタスクをまとめて表示
- カラム上部にアクティブスプリント名を緑ドット付きで表示（複数の場合は `·` 区切り）
- 横スクロール対応
- 各カラム幅 w-52、縦スクロール対応
- カラム間でタスクをドロップすると `status` が変更される
- タスクカードはシンプルな `BacklogTaskCard`（グリップなし・スプリント切替なし）を使用

---

### タスクカード（DraggableTaskCard）

スプリントビュー・エピックビューのリスト表示で使用。

```
[⠿] [○] [PROJ-1  ] [タスクタイトル          ] | [ステータス] [優先度] [エピック] [プロジェクト] [◎n/m] [期限] [▷スプリント名]
```

| 要素 | 説明 |
|------|------|
| ⠿ グリップ | ドラッグハンドル |
| ○ チェック | 完了トグル（クリックで `done` 反転） |
| 課題キー | 固定幅 w-20。クリックでそのタスクのみの詳細ビューへ遷移。課題キーがない場合は空白 |
| タイトル | flex-1 で余白を埋める。完了時は打ち消し線 |
| ステータスバッジ | 色付き小バッジ |
| 優先度バッジ | 色付き小バッジ |
| エピックバッジ | エピック色 + タイトル（最大56px） |
| プロジェクトドット | プロジェクト色ドット + 名前（最大64px） |
| サブタスク数 | `◎ 完了数/総数`（サブタスクがある場合のみ） |
| 期限 | 期限が設定されている場合のみ表示 |
| スプリント切替 | ドロップダウン。現在のスプリント名またはバックログ。完了済みスプリントは非表示 |

- タスクカードをクリック（グリップ・完了チェック・課題キー以外）→ 右パネルに詳細を表示
- スプリント内での課題キー列は固定幅のためタイトルの開始位置が揃う

---

### 右パネル（w-80、タスク詳細ビュー以外で表示）

#### 空の状態（panelMode === "empty"）

「タスクを選択してください」メッセージを表示。

#### タスク作成モード（panelMode === "create"）

新規タスク作成フォーム。

| フィールド | 説明 |
|-----------|------|
| タイトル | テキスト入力（必須） |
| メモ | Markdown エディタ |
| ステータス | セレクト |
| プロジェクト | セレクト |
| スプリント | セレクト（完了済みスプリントは非表示） |
| エピック | セレクト |
| 優先度 | セレクト |
| 期限 | 日付入力 |

ボタン: 「キャンセル」「追加」

#### タスク編集モード（panelMode === "edit"）

選択中タスクの詳細フォーム。**保存ボタンなし・自動保存。**

| フィールド | 保存タイミング |
|-----------|--------------|
| タイトル | onBlur（変更がある場合のみ） |
| メモ（Markdown） | onBlur（変更がある場合のみ） |
| ステータス・プロジェクト・スプリント・エピック・優先度・期限 | onChange（即時） |

フォーム下部:
- サブタスク一覧・追加フォーム
- **コメント / アクティビティ タブ**
- 「このタスクを削除」ボタン

パネルヘッダーに課題キーを表示（クリックで詳細ビューへ遷移）。

---

### タスク詳細ビュー（view === { kind: "task", id }）

右パネルを閉じ、メインコンテンツ全幅を使ってタスク詳細を表示。

```
← 戻る                                    （パンくず: 親タスク / 関連解除リンク）

┌─────────────────────────────────┬─────────────┐
│ # タイトル                        │ ステータス   │
│ ─────────────────────────────── │ 優先度       │
│ 説明（Markdown プレビュー/編集）   │ プロジェクト │
│ ─────────────────────────────── │ スプリント   │
│ サブタスク一覧 / 追加フォーム      │ エピック     │
│ ─────────────────────────────── │ 期限         │
│ 添付ファイル一覧 / 追加            │             │
│ ─────────────────────────────── │ [このタスクを │
│ [コメント (N)] [アクティビティ (N)]│  削除]       │
│ コメント一覧 または アクティビティ  │（sticky）    │
└─────────────────────────────────┴─────────────┘
```

- 左カラム（`flex-1`）: タイトル・説明・サブタスク・添付ファイル・コメントが同幅で縦に並ぶ
- 右カラム（`w-56`、`sticky top-6`）: メタデータサイドバー。スクロールしても追従
- サイドバーの「← 戻る」ではなく、パンくずで前の画面へ戻る
- 親タスクがある場合: パンくずに親タスク名（クリックで親の詳細へ遷移）+「関連解除」リンク
- 自動保存（保存ボタンなし）。動作は右パネルの編集モードと同じ

---

## コメント / アクティビティ タブ

右パネル（edit モード）とタスク詳細ビューの両方に表示。

```
[コメント (N)] [アクティビティ (N)]
```

### コメントタブ

- コメント一覧（作成日時昇順）
- Markdown 記法対応（プレビュー表示）
- 各コメントにホバーで「編集」「削除」ボタン表示
- 編集中: インラインテキストエリア + 「保存」「キャンセル」
- 投稿フォーム: テキストエリア + 「投稿」ボタン（`Ctrl/Cmd + Enter` でも投稿）

### アクティビティタブ

- 変更履歴一覧（新しい順）
- 各エントリの表示形式:
  - `{フィールド}: {旧値（打ち消し線）} → {新値}` （old/new 両方あり）
  - `{フィールド}: {新値}` （追加系）
  - `{フィールド}: {旧値（打ち消し線）}` （削除系）
  - `{フィールド}` のみ （値なし）
- 日時は相対表示（「たった今」「5分前」「2時間前」「3日前」、7日以上は絶対日時）
- タスク選択時・フィールド保存後・コメント操作後・サブタスク操作後に自動リフレッシュ

## プロジェクト画面（mainMenu === "projects"）

ヘッダーの「プロジェクト」タブで切替。全プロジェクトをテーブル形式で一覧表示・管理する。

### テーブル列構成

| 列 | 幅 | 内容 |
|----|-----|------|
| プロジェクト名 | flex | 色ドット + 名前（終了済みは打ち消し線） |
| キー | w-28 | `font-mono` バッジ（未設定は `—`） |
| ステータス | w-28 | `アクティブ`（緑）/ `終了`（グレー）/ `アーカイブ`（アンバー）バッジ |
| タスク数 | w-20 | 親タスク数 |
| 作成日 | w-32 | `YYYY-MM-DD` 表示 |
| アーカイブ日 | w-36 | `archivedAt` 日付（未アーカイブは `—`） |
| 操作 | w-16 | 「編集」ボタン |

### 表示・操作ルール

- デフォルトはアーカイブ済みを非表示。右上「アーカイブ済みを表示/非表示」ボタンで切替
- アーカイブ済み行はグレーアウト（`opacity-60 bg-gray-50`）
- 「編集」ボタン → プロジェクト編集モーダルを開く

### プロジェクト編集モーダル

プロジェクト名変更フォームとステータス操作を一元管理するモーダル。

**フォーム項目:**
- プロジェクト名（テキスト入力）
- キー（英数大文字、最大15文字）
- 「更新」ボタン

**ステータスセクション:**

| 現在の状態 | 表示するボタン |
|-----------|--------------|
| active・非アーカイブ | 「終了する」（オレンジ）/ 「アーカイブ」（アンバー） |
| closed | 「再開する」（グリーン）/ 「アーカイブ」（アンバー） |
| アーカイブ済み | 「アーカイブ解除」（グレー） |

- **終了する**: `status` を `closed` に変更し、同時に `archivedAt` を現在時刻に設定（終了 = 自動アーカイブ）
- **再開する**: `status` を `active` に変更
- **アーカイブ**: `archivedAt` を現在時刻に設定
- **アーカイブ解除**: `archivedAt` を null に設定

**削除操作:**
- セクション下部に「このプロジェクトを削除」ボタン（赤、全幅）
- クリック → `DELETE /api/projects/[id]`（タスクの `projectId`・`taskNumber` を null にして削除）

## スプリント画面（mainMenu === "sprints"）

ヘッダーの「スプリント」タブで切替。サブメニュー「スプリント管理」「スプリント記録」をタブで切替。

---

### スプリント管理（sprintSubMenu === "management"）

テーブル形式でスプリントを一覧表示・インライン編集。

#### テーブル列構成

| 列 | 幅 | 内容 | 編集方法 |
|----|----|------|---------|
| スプリント名 | w-48 | スプリント名 | テキスト入力（onBlur で即時保存） |
| ステータス | w-24 | `planning` / `active` / `completed` | セレクト（onChange で即時保存）。`planning` / `active` のみ選択可 |
| 開始日 | w-28 | 開始日 | 日付入力（onChange で即時保存） |
| 終了日 | w-28 | 終了日 | 日付入力（onChange で即時保存） |
| タスク数 | w-20 | `完了数 / 総数`（親タスクのみカウント） | 読み取り専用 |
| アクション | w-24 | 「完了」「削除」ボタン | — |

#### 表示・操作ルール

- **表示順:** planning → active → completed（completed はクローズ日時の新しい順）
- デフォルトは completed スプリントを非表示。右上の「クローズ済みを表示 / 非表示」ボタンで切替
- completed スプリント行はグレーアウト・読み取り専用
- 「完了」ボタンは completed 以外のスプリントのみ表示
- スプリント総件数を左上に表示

---

### スプリント完了モーダル

「完了」ボタン押下で表示。

**表示内容:**

1. スプリント名（見出し）
2. 未完了タスク数の確認
   - 対象: `done === false` かつ `status !== "closed"` の親タスク
   - 0件の場合: 「すべてのタスクが完了しています。」
   - 1件以上の場合: 件数（オレンジ色）と移動先選択ドロップダウン
3. 移動先選択肢: バックログ、または completed 以外の他スプリント
4. 「キャンセル」「完了する」ボタン

**完了処理（サーバーサイド /api/sprints/[id]/complete POST）:**

1. `done === false` かつ `status !== "closed"` のタスク → 選択した移動先へ `sprintId` 変更
2. `status === "closed"` のタスク → スプリントを変更しない（元のスプリントに残す）
3. スプリント内の全親タスクのスナップショットを `SprintTaskRecord` に保存
4. スプリントの `status` を `completed`、`closedAt` に現在時刻を設定

### スプリント記録（sprintSubMenu === "records"）

2ペイン構成。

#### 左ペイン（w-64）: スプリント一覧

表示対象: `status === "active"` または `status === "completed"` のスプリント

表示順: アクティブ → 完了済み（closedAt の新しい順）

各行:
- カラードット（アクティブ: 緑、完了済み: グレー）
- スプリント名
- クローズ日時（完了済みの場合のみ）

#### 右ペイン: タスク記録詳細

| 状態 | 表示内容 |
|------|---------|
| 未選択 | 「左のスプリントを選択してください」 |
| 読み込み中 | 「読み込み中…」 |
| アクティブスプリント | 「アクティブなスプリントです。クローズ後に記録が保存されます。」バナー |
| 完了済みスプリント | タスク記録一覧 |

**完了済みスプリント選択時の表示:**

- スプリント名・開始日・終了日・クローズ日時
- サマリー: `完了 N | 未完了 N | 合計 N`

**完了タスク** (`wasDone === true`):
- 緑のチェックアイコン ＋ タイトル（打ち消し線）
- クリック: タスク画面に切替 + タスク詳細ビューへ遷移（タスクが存在する場合）
- タスク削除済みの場合: 「削除済み」ラベル、クリック不可

**未完了・移動したタスク** (`wasDone === false`):
- オレンジの × アイコン ＋ タイトル
- 移動先スプリント名またはバックログ表示
- クリック: タスク画面に切替 + タスク詳細ビューへ遷移（タスクが存在する場合）
- タスク削除済みの場合: 「削除済み」ラベル、クリック不可

### レイアウト（二カラム）

## 管理モーダル（⚙）

グローバルヘッダー右端の歯車アイコンで開くモーダル。以下の機能を持つ。

### アップデート

- モーダルを開いたとき `/api/admin/version` を自動フェッチし、現在のコミットハッシュ・日付・コミットメッセージを表示
- 「アップデートを確認」ボタンで `/api/admin/update` を呼び出し
- アップデート前に DB バックアップを自動作成（`prisma/backups/dev.db.{hash}.{YYYYMMDDHHMMSS}`）
- 各ステップ（git fetch / DB バックアップ / git pull / npm install / prisma db push）の結果をログ表示
- 最新の場合は「最新のバージョンです」メッセージを表示

### ロールバック

- モーダルを開いたとき `/api/admin/backups` を自動フェッチし、バックアップ一覧を表示
- 各バックアップ行: バッジ（アップデートバックアップはコミットハッシュ、自動バックアップは「自動」）・タイムスタンプ・ファイルサイズ・「復元」ボタン
- 「復元」クリック → 確認ダイアログ → `/api/admin/rollback` を呼び出し
  - アップデートバックアップ: `git reset --hard {hash}` + DB 復元 + `prisma generate`
  - 自動バックアップ: DB 復元のみ（コードは変更しない）+ `prisma generate`
- 完了後「サーバーを再起動してください」メッセージを表示

---

## 自動バックアップ

`instrumentation.ts`（Next.js サーバー起動時に実行）で `node-cron` を使い、**毎日午前 3 時**にバックアップを自動作成する。

- バックアップファイル名: `prisma/backups/dev.db.auto.{YYYYMMDDHHMMSS}`
- ローテーション: `dev.db.auto.*` を新しい順に **3 件** まで保持し、超過分を自動削除
- アップデート時に作成されるバックアップ（`dev.db.{hash}.{ts}`）は対象外
- `prisma/backups/` は `.gitignore` で除外（コミット不可）
