# タスク管理アプリケーション 仕様書

## 1. 概要

シンプルなタスク管理ウェブアプリケーション。ローカル環境で動作し、タスクの作成・編集・削除・完了管理を行う。

---

## 2. 画面・ビュー一覧

### 画面

| 画面名 | URL | 説明 |
|--------|-----|------|
| タスク一覧画面 | `/` | タスクの一覧表示・詳細確認・作成・編集・削除を行うメイン画面 |

### ビュー（タスク一覧画面内の構成要素）

| ビュー名 | 位置 | 説明 |
|---------|------|------|
| スプリントビュー | 左 | バックログ・スプリントの切り替えナビゲーション、スプリントの作成・削除 |
| タスクリストビュー | 中央 | 選択中のスプリント／バックログのタスク一覧、フィルター操作 |
| タスク詳細ビュー | 右 | 選択タスクの詳細表示、タスクの作成・編集フォーム |

### Sprint テーブル

| フィールド | 型 | 必須 | デフォルト | 説明 |
|-----------|-----|------|-----------|------|
| id | Int | ✓ | autoincrement | 主キー |
| name | String | ✓ | — | スプリント名 |
| startDate | DateTime | — | null | 開始日 |
| endDate | DateTime | — | null | 終了日 |
| status | String | ✓ | "planning" | ステータス（planning / active / completed） |
| createdAt | DateTime | ✓ | now() | 作成日時 |

### Task テーブル

| フィールド | 型 | 必須 | デフォルト | 説明 |
|-----------|-----|------|-----------|------|
| id | Int | ✓ | autoincrement | 主キー |
| title | String | ✓ | — | タスクのタイトル |
| description | String | — | null | メモ・詳細 |
| done | Boolean | ✓ | false | 完了フラグ |
| priority | String | ✓ | "medium" | 優先度（low / medium / high） |
| dueDate | DateTime | — | null | 期限日 |
| sprintId | Int | — | null | 所属スプリントID（null = バックログ） |
| createdAt | DateTime | ✓ | now() | 作成日時 |

## 6. 機能一覧

### 6.1 スプリントナビゲーション（スプリントビュー）

- 「バックログ」と各スプリントをボタンで切り替える
- 選択中のビューはインジゴ色でハイライトされる
- 各項目の右にタスク件数を表示する
- スプリントには名前・ステータスバッジ・期間を表示する
- スプリントにカーソルを当てると削除ボタン（×）が現れる

### 6.2 スプリント作成（スプリントビュー）

- サイドバーの「+ 追加」ボタンで作成フォームをインライン表示する
- 入力項目
  - スプリント名（必須）
  - 開始日・終了日（任意）
  - ステータス（計画中 / アクティブ / 完了、デフォルト: 計画中）
- 作成後、新しいスプリントのビューに自動遷移する

### 6.3 スプリント削除（スプリントビュー）

- サイドバーのスプリント行にカーソルを当てると表示される × ボタンをクリック
- 確認ダイアログを表示してから削除する
- 削除時、スプリント内のタスクは自動的にバックログへ移動する
- 削除したスプリントが選択中だった場合、バックログビューへ遷移する

### 6.4 タスク一覧表示（タスクリストビュー）

- 選択中のビュー（バックログ or スプリント）のタスクを作成日時の降順で表示する
- 各タスクカードに以下を表示する
  - タイトル（完了時は取り消し線）
  - 優先度バッジ（色分け）
  - メモ（1行に省略表示）
  - 期限日
- カードをクリックすると右パネルに詳細を表示する
- 選択中のカードはインジゴ色のボーダーでハイライトされる

### 6.5 フィルター（タスクリストビュー）

- 「すべて」「未完了」「完了済み」の3種類で表示を切り替える
- 現在表示中の件数を右端に表示する

### 6.6 タスク選択（タスク詳細ビュー: 編集フォームモード）

- タスクカードをクリックすると、タスク詳細ビューに即座に編集フォームを表示する
- 既存の値（タイトル・メモ・スプリント・優先度・期限）がフォームに入った状態で開く
- 「更新」ボタンで保存後、フォームは最新の値で更新されたまま編集モードを維持する
- 「キャンセル」ボタンで選択を解除し、空状態に戻る
- フォーム下部の「このタスクを削除」で確認ダイアログを表示し削除する
- 削除後、タスク詳細ビューは空状態に戻る

### 6.7 タスク作成（タスク詳細ビュー: 作成フォームモード）

- ヘッダーの「+ タスクを追加」ボタンでタスク詳細ビューを作成フォームに切り替える
- 入力項目
  - タイトル（必須）
  - メモ（任意）
  - スプリント（バックログ / 各スプリント、現在のビューをデフォルト値とする）
  - 優先度（低 / 中 / 高、デフォルト: 中）
  - 期限（任意）
- 「追加」ボタンで保存後、作成したタスクの編集フォームに切り替える
- 「キャンセル」ボタンで空状態に戻る
- タイトルが空の場合は送信しない

### 6.8 完了・未完了の切り替え

- タスクカード左端の丸ボタンをクリックで `done` を反転する（タスクリストビューから操作可）
- 完了状態: ボタンがインジゴ色に塗りつぶされ、タイトルに取り消し線が入る

## 8. API 仕様

### GET /api/tasks

全タスクを取得する。

**レスポンス** `200 OK`
```json
[
  {
    "id": 1,
    "title": "タスク名",
    "description": "メモ",
    "done": false,
    "priority": "medium",
    "dueDate": "2026-07-01T00:00:00.000Z",
    "sprintId": 1,
    "createdAt": "2026-06-27T14:00:00.000Z"
  }
]
```

---

### POST /api/tasks

タスクを新規作成する。

**リクエストボディ**
```json
{
  "title": "タスク名",       // 必須
  "description": "メモ",    // 任意
  "priority": "medium",     // 任意（low / medium / high）
  "dueDate": "2026-07-01",  // 任意
  "sprintId": 1             // 任意（null = バックログ）
}
```

**レスポンス** `201 Created` — 作成されたタスクオブジェクト

---

### PATCH /api/tasks/:id

タスクを部分更新する。指定したフィールドのみ更新される。

**リクエストボディ**（すべて任意）
```json
{
  "title": "新しいタイトル",
  "description": "新しいメモ",
  "done": true,
  "priority": "high",
  "dueDate": "2026-08-01",
  "sprintId": 2
}
```

**レスポンス** `200 OK` — 更新後のタスクオブジェクト

---

### DELETE /api/tasks/:id

タスクを削除する。

**レスポンス** `204 No Content`

---

### GET /api/sprints

全スプリントを作成日時の昇順で取得する。

**レスポンス** `200 OK`
```json
[
  {
    "id": 1,
    "name": "Sprint 1",
    "startDate": "2026-07-01T00:00:00.000Z",
    "endDate": "2026-07-14T00:00:00.000Z",
    "status": "active",
    "createdAt": "2026-06-27T14:00:00.000Z"
  }
]
```

---

### POST /api/sprints

スプリントを新規作成する。

**リクエストボディ**
```json
{
  "name": "Sprint 1",          // 必須
  "startDate": "2026-07-01",   // 任意
  "endDate": "2026-07-14",     // 任意
  "status": "planning"         // 任意（planning / active / completed）
}
```

**レスポンス** `201 Created` — 作成されたスプリントオブジェクト

---

### PATCH /api/sprints/:id

スプリントを部分更新する。

**リクエストボディ**（すべて任意）
```json
{
  "name": "Sprint 1 改",
  "status": "active",
  "startDate": "2026-07-01",
  "endDate": "2026-07-14"
}
```

**レスポンス** `200 OK` — 更新後のスプリントオブジェクト

---

### DELETE /api/sprints/:id

スプリントを削除する。削除前にスプリント内の全タスクの `sprintId` を `null`（バックログ）に更新する。

**レスポンス** `204 No Content`

## 9. UI 仕様

### 優先度の色

| 優先度 | バッジカラー |
|--------|------------|
| low（低） | 青（bg-blue-100 / text-blue-700） |
| medium（中） | 黄（bg-yellow-100 / text-yellow-700） |
| high（高） | 赤（bg-red-100 / text-red-700） |

### スプリントステータスの色

| ステータス | バッジカラー |
|-----------|------------|
| planning（計画中） | グレー（bg-gray-100 / text-gray-600） |
| active（アクティブ） | 緑（bg-green-100 / text-green-700） |
| completed（完了） | 紫（bg-purple-100 / text-purple-700） |

### タスク詳細ビューのモード遷移

| 操作 | 遷移先 |
|------|--------|
| タスクカードをクリック | 編集フォームモード |
| 「+ タスクを追加」クリック | 作成フォームモード |
| 編集フォームで「更新」 | 編集フォームモード（最新値で維持） |
| 編集フォームで「キャンセル」 | 空状態 |
| 作成フォームで「追加」 | 編集フォームモード（新規タスク） |
| 作成フォームで「キャンセル」 | 空状態 |
| 「このタスクを削除」→ 確認OK | 空状態 |

### フォーム共通

- 作成・編集フォームはモーダルを使用せず、タスク詳細ビュー内にインラインで表示する
- フォームを開いた際にタイトル入力欄に自動フォーカスする

## 11. 起動方法

```bash
# 依存パッケージのインストール
npm install

# DB マイグレーション（初回のみ）
npx prisma migrate dev --name init

# 開発サーバー起動
npm run dev
```
